# Security

## Guidance:

- Never directly execute any code assistant on the developer host machine. Every single code agent action should be executed by running a Docker image that has the agent tool installed and passing an entrypoing command to direct the agentic tool.
  - **Sole exception — the `ready` local-agent check** (`ReadyPhase::CheckingLocalAgent` in `src/engine/ready/mod.rs`): during `awman ready`, the configured agent's binary IS executed once on the host with a fixed, innocuous greeting prompt. This is deliberate and required: it verifies the host-side agent is installed and authenticated, and it causes the agent to refresh its auth token so that fresh credentials are what get mounted/injected into the containerized agents. This exception is strictly scoped — the prompt is a hardcoded greeting chosen by awman (never user input, repo content, or task instructions), no working directory or repo data is passed to the agent, and no other code path may execute an agent on the host. Any new host-side agent invocation beyond this check is a security violation.
- Never mount any directory to any Docker container other than the current directory. If any parent directories are a Git repo root, the aspec CLI will prompt the user if the mounted directory should be limited to the current CWD or can be expanded to the Git repo root. Follow this instruction for every single container launched.
